# Explore Topic：从“抄作业”到可检验的设计

## 一句话叙事

我们没有声称从零发明一种 AI 研究方法，而是把信息觅食、动态视角、延迟收敛、反证搜索等已有机制压缩成一个薄 Skill，再检验它能否在不显著增加时间和篇幅的前提下，扩大开放研究的探索空间。

## 为什么从厚流程转向薄 Skill

早期目标是证明一套完整研究流程能比模型默认研究产生更强洞察。连续对照暴露了更具体的问题：厚 Skill 能改善边界、可追溯性和验证意识，但也容易让模型过早服从输出结构，在相似的“变化—矛盾—机会—验证”栏目里重复同一发现。结果可能更规整，却未必发现更多。

因此，当前问题不是“怎样让报告更像一份专业报告”，而是：

> 能否只加入少数认知动作，让模型在综合之前看见更多不同解释，并主动离开第一个看似合理的框架？

这也是新旧两个 Skill 的分工：

- `explore-topic`：开放探索，优先扩大视角、发现异常、保留替代解释；
- `decision-research`：严谨综合、方案比较、结论审计与决策输入。

前者不是后者的精简版，而是优化目标不同的另一种工作模式。

## 借鉴了什么

### 1. 信息觅食与意义建构

[Pirolli 与 Card 的 sensemaking loop](https://www.researchgate.net/publication/215439203_The_sensemaking_process_and_leverage_points_for_analyst_technology_as_identified_through_cognitive_task_analysis) 区分了搜集信息和形成结构。我们据此把“侦察”和“综合”分开：先让证据改变问题地图，再写结论，而不是边搜边填预设报告结构。

### 2. 动态视角与可见的问题地图

[STORM / Co-STORM](https://github.com/stanford-oval/storm) 将写作前研究、不同视角的问题生成、知识组织和用户介入分开。新 Skill 借鉴的是机制：视角应随证据生成，问题之间的关系应持续更新；没有照搬其系统架构或品牌化流程。

### 3. 先宽后窄、按价值分配搜索

[Anthropic 的多智能体研究系统](https://www.anthropic.com/engineering/multi-agent-research-system) 强调先广泛探索，再沿高价值分支深入，并对研究投入进行预算。新 Skill 把这压缩成单智能体也能执行的启发式：按相关性、解释力、新颖性和证据前景选分支，而不是按最容易搜索的方向深入。

### 4. 反证与独立证据

科学研究和调查研究都要求主动寻找会推翻当前解释的证据。新 Skill 要求至少检查一个可信的反证分支和一个被忽略或相邻的视角，同时区分厂商主张、观察到的行为、运营现实与最终结果。

### 5. 把 Skill 当作可迭代干预

[SkillOpt](https://arxiv.org/abs/2605.23904) 提供了更直接的方法论先例：把 Skill 作为可版本化的外部文本状态，通过有评分的运行、有限修改和留出任务验证来优化。我们据此不把一次好结果当成证明，也不在每轮失败后无限加规则。

### 6. 防止局部精耕代替广泛探索

2026 年预印本 [AI Research Agents Narrow Scientific Exploration](https://arxiv.org/abs/2605.27905) 把研究代理视为搜索系统；其科学创意实验发现，代理生成的方向更集中，也更靠近起始文献。这个结果不能直接外推到产品研究，却为我们的设计风险提供了可检验假设：模型可能擅长沿首个框架继续精耕，却不自然地扩大问题空间。因此，新 Skill 不只要求“多想几个角度”，还要求检查相邻视角、反证分支和未解异常。

## 我们真正贡献什么

原创性不在于发明“多视角”或“反证”这些概念，而在于四件更诚实、也更容易检验的事：

1. **选择**：从大量研究方法中选出最可能缓解开放研究过早收敛的机制；
2. **压缩**：把这些机制缩成模型可执行、不会吞噬任务本身的短指令；
3. **操作化**：把抽象原则写成可观察动作，如动态问题图、打破首个框架、地图饱和停止；
4. **评估**：用同模型、同题目、同工具的对照实验判断收益和成本，而不是只展示最好的一篇报告。

作品集里可以把这段过程命名为：

> **从自创流程到基于先例的设计：将研究理论与研究代理机制压缩为一个可验证的薄 Skill。**

“抄作业”在这里不是隐藏来源或复制实现，而是承认成熟先例、提取机制、说明取舍，再用自己的场景验证它是否成立。

## 当前假设

薄 Skill 不保证让每篇报告都更长、更完整或立刻产生产品方向。它首先应当带来以下可观察变化：

- 出现更多实质不同的视角，而不是同一发现的多种命名；
- 至少有一个分支挑战最初的主导解释；
- 能保留一个有证据基础的替代解释或未解异常；
- 机会假设来自机制与矛盾，而不是栏目要求；
- 当公开网页趋于饱和时，指出下一步需要何种非网页证据；
- 时间和输出长度的增量显著低于厚 Skill。

## 下一轮怎样验证

建议做三条件对照：

| 条件 | 工作流 | 作用 |
| --- | --- | --- |
| A | 无 Skill | 模型默认基线 |
| B | `explore-topic` 薄 Skill | 检验探索机制的净增益 |
| C | `decision-research` 厚 Skill | 比较探索广度与研究纪律的取舍 |

三组固定同一模型、reasoning effort、自然语言 prompt、工具权限和停止条件。每次记录时间、输出长度和人工审阅时间。

重点评分：

- `perspective_breadth`：实质不同的视角数量与跨度；
- `frame_break`：是否有证据改变或限制首个解释；
- `novel_branch_count`：后续搜索中新出现且有价值的分支；
- `semantic_redundancy`：发现之间是否只是改名重复；
- `strong_insight_count`：达到“机制 + 产品含义”或更高等级的洞察数；
- `counterevidence`：是否搜索并公平处理反证；
- `source_quality` 与 `freshness_scope`：来源质量及时空边界；
- `next_evidence_value`：建议的下一步证据能否区分竞争解释。

智能手环、具身智能以及已经用于修改 Skill 的题目都属于开发集。下一步至少选三个行业的未见题目作为留出集，并避免只挑适合公开网页研究的主题。一次新题目的胜负只能形成线索，不能证明普遍增益。

## 不应声称的内容

- 不声称新 Skill 全面优于 ChatGPT 或其他研究产品；
- 不把更长、更整齐或引用更多等同于更有洞察；
- 不用开发集成绩证明跨行业泛化；
- 不把前人已有的研究机制包装成独创理论；
- 不因薄 Skill 在探索上胜出，就推断它更适合固定选项决策或高风险审计。
