# Explore Topic

Explore before explaining. Treat the topic as an evolving map, not as a report outline to fill.

## 1. Set a light boundary

Infer the user's purpose, primary geography or population, time horizon, and available effort. State only assumptions that materially affect the research. Do not invent a decision, fixed options, or a recommendation.

## 2. Scout before forming conclusions

Begin with short, broad searches and current direct sources. Let the first useful evidence generate several materially different perspectives rather than applying a fixed industry checklist.

Make perspectives differ by actor, mechanism, system level, time period, geography, or competing explanation. Merge perspectives that would produce the same searches. Do not draft opportunities yet.

## 3. Maintain an issue map

As evidence arrives, keep a lightweight working map of:

- important changes and boundary conditions;
- actors with different incentives or experiences;
- tensions, contradictions, and measurement mismatches;
- anomalies that do not fit the current explanation;
- plausible explanations and alternatives;
- unknowns and weakly explored branches.

Attach sources and scope to consequential nodes. Update relationships when the frame changes. The map is working state; show it only when it helps the user steer or understand the result.

## 4. Break the first frame

Before narrowing, ask:

- What evidence would make the current frame wrong or incomplete?
- Which actor is missing?
- Does an adjacent category suggest a different mechanism?
- Have region, version, population, time, or denominator differences been flattened?

When relevant, pursue at least one credible disconfirming branch and one neglected or adjacent perspective. Prefer evidence independent of the dominant claim's origin.

## 5. Deepen selected branches

Choose branches by user relevance, explanatory value, novelty, and evidential promise, not merely by search convenience. Search directly for support and counterevidence. Distinguish vendor promises, observed behavior, operational reality, and outcomes.

Pause for user steering only when a branch would materially redefine the task. Otherwise continue with the best-supported interpretation and preserve meaningful alternatives.

## 6. Synthesize late

Match the artifact and language to the request. Keep the output proportional to the evidence.

- Separate observation, interpretation, and opportunity hypothesis.
- Cite consequential claims and preserve their population, geography, version, denominator, and date boundaries.
- State a mechanism only when evidence supports it; otherwise label it as a hypothesis.
- For a promising opportunity, explain why it is non-obvious and name the cheapest evidence that would distinguish it from the strongest alternative explanation.
- Preserve at least one material alternative or unresolved anomaly.
- Do not repeat the same finding as a change, tension, opportunity, and test.
- Do not invent thresholds, sample sizes, confidence, or false precision.

Do not impose a fixed card count, ledger, recommendation, or report template unless the user asks for one.

## 7. Stop on map saturation

Stop when new searches mostly reinforce the map without adding a perspective, changing a relationship, or resolving a material uncertainty. If public webpages have saturated, identify the behavioral, operational, user, or experimental evidence needed next instead of extending the report with more summaries.
