# Decision Research v0.2：从对照结果到有界优化

> 记录日期：2026-07-19  
> 证据等级：探索性开发结论，不是稳定泛化证明  
> 用途：保留 Skill 优化依据，并为作品集案例提供可审计的过程素材

## 1. 最初假设

同一个模型使用 Decision Research Skill 后，应该减少研究错误，并产生更多可追溯、可反证、能改变产品判断的洞察。自由研究中的“强洞察”定义为深度等级达到 3 或 4：不仅识别变化或矛盾，还提出可能机制、产品含义，或进一步改变优先级并给出验证和推翻条件。

## 2. 两轮观察

### 开发轮 A：AI 会议助手

| 指标 | 无 Skill | 有 Skill |
|---|---:|---:|
| elapsed time | 3 min | 9 min |
| strong insights | 4 | 4 |
| top-3 depth mean | 3.67 | 4.00 |
| output characters | 9,884 | 12,979 |
| unique URLs / domains | 19 / 10 | 11 / 7 |

Skill 提高了追溯、反证、测试和更新能力，但没有增加强洞察数量；输出更长，搜索覆盖更窄，并把自由发现题强行包装成 Decision Brief。两份原始输出已于 2026-07-20 找回并归档，可独立复核；上表字符数已按找回文件的统一 UTF-8 读取方式更正，URL/域名统计与原记录一致。

### 开发轮 B：智能手环，自然语言隐式触发

实际提示：

> 帮我研究一下智能手环。我想知道它发生了哪些重要变化、用户实际使用中有哪些矛盾，以及有哪些不太显然但值得继续验证的机会。不要预设结论，请附上来源。不要读取当前工作区之外的目录。

| 指标 | 无 Skill | 有 Skill |
|---|---:|---:|
| elapsed time | 2 min | 3 min |
| valid / strong insights | 5 / 5 | 4 / 4 |
| top-3 depth mean | 3.33 | 4.00 |
| output characters | 6,744 | 11,901 |
| unique URLs / domains | 13 / 11 | 16 / 9 |

有 Skill 产物完整出现 Decision Brief、Research Snapshot、Claim-Evidence Ledger、Counterevidence、Tests、Recommendation 和 Stop/Update Record，可高置信判断自然语言隐式触发成功。其洞察更容易验证和推翻，但四个方向中三个与无 Skill 产物重合；无 Skill 产物反而覆盖了额外的隐私授权和商业模式机会。

本轮有 Skill 仍较慢，但差距从开发轮 A 的 3 倍缩小到 1.5 倍。由于样本各只有一次、时间为人工分钟级记录，目前只能说两轮都观察到额外延迟，不能估计稳定幅度。

## 3. 初步判断

当前证据支持以下窄结论：

1. **L0 部分成立：** Skill 能在两个领域生成完整、可读、可审计的研究结构，并能被自然语言隐式触发一次。
2. **发现能力尚未证明：** 两轮都没有增加强洞察数量，也没有稳定扩大来源或机会覆盖。
3. **后处理能力有重复信号：** Skill 更稳定地补充机制、反证、判别测试、更新条件和置信度。
4. **稳定失败是模式错配：** 没有真实决策时仍虚构负责人、选项和推荐，造成格式负担并提前收敛。
5. **成本方向有初步信号：** 两轮有 Skill 都更慢，但幅度差异很大；篇幅负担更稳定，有 Skill 产物分别长约 36% 和 76%。

因此，当前 Skill 更像“Insight → Decision 后处理器”，尚未证明是更强的自由发现器。

## 4. v0.2 有界修改

只修改一个相互关联的机制：**先选择研究模式，再选择输出契约。**

| 重复失败 | v0.2 干预 | 预期可观察变化 |
|---|---|---|
| 自由研究被强制写成决策题 | 增加 Discovery / Decision / Audit-update 路由 | Discovery 不再虚构负责人、选项和推荐 |
| 同一判断在多张表中重复 | Discovery 默认不展示完整台账和停止更新表 | 篇幅下降，证据仍邻近洞察 |
| 格式占用搜索与发现预算 | 明确先搜索和挑战关键主张，再格式化 | 来源和机会覆盖不因模板提前收敛 |
| 中文请求出现英文字段 | 用户可见标题和表格跟随提示语言 | 中文报告不再出现机器字段式混排 |
| 判别测试出现任意数字门槛 | 禁止无基线、决策规则或证据依据的数值门槛 | 测试条件更诚实、可解释 |

没有修改来源政策、事实—推断—假设区分和反证要求，因为这三项是两轮中最稳定的正向信号。

## 5. v0.2 接受条件

先在开发任务上确认基本行为，再用未见过的行业做 holdout：

- Discovery 不出现虚构的 Decision Brief、负责人、A/B/C/D 或唯一推荐；
- 中文自然语言请求的用户可见字段保持中文；
- 默认输出 3–5 条不重复洞察，`top3_depth_mean >= 3.5`；
- 关键事实、引用准确性、范围和置信度门槛不低于 v0.1；
- Discovery 默认不展示完整 claim ledger；
- 多次运行后，有 Skill 产物的中位篇幅不超过无 Skill 条件的 1.4 倍；
- 在至少一个新行业中，Skill 的改善仍可观察，才把结论从开发信号升级为泛化证据。

## 6. 作品集叙事结构

网站案例可以按以下顺序呈现：

1. **问题：** 通用研究流程是否真的比默认模型更有价值？
2. **操作化：** 把“强洞察”定义为可评分的 0–4 层级，并把事实错误设为独立门槛。
3. **实验：** 同模型、同提示、隔离工作区，对照有无 Skill；再测试自然语言隐式触发和跨行业迁移。
4. **意外结果：** Skill 没有发现更多洞察，只把洞察加工得更可验证；时间成本也不稳定。
5. **诊断：** 固定 Decision Brief 使开放发现过早收敛，并制造结构和语言负担。
6. **干预：** 引入研究模式路由和按需输出，而不是继续增加模板。
7. **下一步：** 用开发任务回归，再以新行业、多次重复和小模型条件验证。

这个案例的价值不应包装成“已经证明 Skill 胜过 ChatGPT”，而是展示如何把模糊的研究质量转成可测假设，并让失败结果驱动一次可验证的工作流迭代。

## 7. 当前限制与记录缺陷

- 每个条件、每个题目只有一次运行，不能估计随机波动；
- 第二轮评审者知道工作区映射，不是严格盲评；
- 运行时间为人工分钟级记录，没有 token 和成本数据；
- 第二轮最初复用了同名路径；第一轮原始报告后来从执行工作区找回并单独归档，第二轮也已迁移到独立路径；
- 工作区内重复的 `TASK.md`、`START-HERE.md` 和 `tasks/` 已删除，实际提示统一保存在实验根目录 `prompts/`；旧 claim ledger 已移入 `evaluation/archive`；
- 两个已用于 v0.2 修改的题目都应标记为 development data，不再充当 holdout。

## 8. 实施状态

v0.2 已于 2026-07-19 写入 `decision-research`，并通过 `skill-creator/scripts/quick_validate.py`。本次修改限于模式路由、输出契约、语言规则和对应评估门槛；来源政策、评分脚本与核心事实核查原则未改。

用于隔离对照的 `research-beta/.agents/skills/decision-research` 已同步同一版本并单独通过验证；顶层源文件与 Beta 副本的 `SKILL.md` 哈希一致。`research-alpha` 保持无 Skill。

Skill 目录的写入拒绝来自父级 `.agents`，不是 Skill 自身配置。实施期间仅对 `decision-research` 子树临时移除两个 Deny SID；写入与验证完成后，已通过 ACL reset 恢复为完全继承父目录规则，未改变其他 Skill 或 `.agents` 的保护。
