## Skill 元数据

**名称：** `decision-research`

**触发描述：**

把开放式的产品、市场、行业、技术或政策研究，转化为时效明确、可追溯、可反证、可更新的决策输入。

当 Codex 需要完成以下任务时使用：

- 调查一个存在多个竞争选项的决策；
- 审计或更新已有研究结论；
- 区分事实、推断和假设；
- 建立主张—证据账本；
- 主动寻找反证；
- 形成与决策相关的洞察；
- 比较不同研究工作流或模型。

不要用于简单、稳定的事实查询，也不要用于不涉及研究决策的普通写作任务。

# 决策研究

把研究从一篇表面完整的叙事，转化为一个可审计的决策过程。优化目标是：改变了什么决定、暴露了什么错误、减少了什么不确定性，而不是篇幅或引用数量。

## 运行规则

- 在大范围搜索前先定义决策。如果没有任何决策会因研究而改变，明确把任务降级为背景资料整理。
- 把“主张”而不是“网页”作为研究单位。
- 只要时间、版本、地区、人群和测量口径可能改变含义，就必须记录它们。
- 把每条重要陈述标记为 `事实（fact）`、`推断（inference）` 或 `假设（hypothesis）`。
- 在强化结论之前，主动搜索反对证据和替代解释。
- 优先使用当前的一手证据。多个网页重复同一个上游来源，不能伪装成相互独立的交叉验证。
- 让置信度和表达语气与证据强度匹配。
- 当继续搜索的价值低于获取用户、行为、运营或实验数据的价值时停止搜索。
- 默认先使用单 Agent。只有当独立研究分支能够明显减少上下文污染或延迟时，才增加并行 Agent，并记录额外成本。

## 工作流

### 1. 定义决策

说明谁来决策、截止时间、真实可选方案、约束、错误决策成本，以及哪些证据可能改变选择。除非存在至少两个真实选项，否则不要把任务继续包装成决策研究。

### 2. 固定研究快照

记录资料截止日期、地区、产品及版本、目标人群、比较口径、允许使用的来源、预算和预计有效期。不要混合无法兼容的范围。

### 3. 建立主张—证据账本

每个可被外部核验的主张单独占一行。直接链接证据，记录证据日期和来源等级，并把支持关系标为直接、间接或冲突。

产物字段见 [研究模板](../.agents/skills/decision-research/references/templates.md)。可以使用 [`validate_claim_ledger.py`](../.agents/skills/decision-research/scripts/validate_claim_ledger.py) 做确定性检查。

### 4. 检查时效与反证

对每条影响决策的关键主张：

1. 寻找更新的一手来源；
2. 搜索可能推翻它的证据；
3. 列出至少一个合理的替代解释；
4. 检查看似独立的来源是否共享同一个上游材料；
5. 区分厂商承诺、实际观察到的行为和最终业务结果。

选择或审计来源时，参考 [来源及时效规范](../.agents/skills/decision-research/references/source-policy.md)。

### 5. 形成洞察

使用固定结构：

`观察到的变化 → 可能机制 → 对决策的影响 → 证伪条件`

只有当一条判断超越材料摘要，并且改变了方案、优先级、风险判断或验证路径时，才把它称为洞察。没有机制时只能称为观察；没有行为或因果证据时，避免使用因果语言。

### 6. 选择最低成本的区分性验证

提出能够区分至少两个竞争解释的最低成本证据：

- 验证功能是否存在：检查当前产品或版本；
- 验证用户是否遇到问题：查看行为日志或进行任务观察；
- 验证方案是否容易理解：使用原型；
- 验证原因机制：进行对照测试；
- 验证业务结果：进行小规模试点。

### 7. 停止研究并支持后续更新

记录当前证据能支持和不能支持的判断、风险最高的剩余假设、停止搜索的原因、结论失效日期、更新触发条件，以及下一位研究者如何继续。

新证据出现时，更新受影响的账本行及其下游决策，不要默认重写整份报告。

## 必需输出

除非用户要求更窄的产物，否则输出：

1. 决策简报（Decision Brief）
2. 研究快照（Research Snapshot）
3. 主张—证据账本（Claim-Evidence Ledger）
4. 反对证据与替代解释
5. 决策洞察
6. 最低成本区分性验证
7. 包含置信度和适用边界的建议
8. 停止与更新记录

使用 [研究模板](../.agents/skills/decision-research/references/templates.md) 中的紧凑格式。任何未通过的质量门槛都必须直接暴露，不能埋在叙述中。

## 质量门槛

- **G1 当前且正确：** 影响决策的关键事实符合既定时间、版本、地区和测量口径。
- **G2 可追溯：** 每条关键结论都能映射到具体证据。
- **G3 与决策相关：** 研究改变了方案、优先级、风险判断或下一步验证。
- **G4 可反证：** 反对证据、替代解释、不确定性和证伪条件都被明确记录。
- **G5 经过现实验证：** 用户、行为、运营或实验数据支持结论。

通过 G1–G4 但没有通过 G5 的产物，只能称为**可测试的研究假设**，不能称为已经验证的产品或业务结论。

## 评测模式

评估这个 Skill 是否有效时，必须先阅读 [评测协议](../.agents/skills/decision-research/references/evaluation-protocol.md)，再开始任何实验。

固定以下条件：

- 提示词；
- 工具与权限；
- 时间和成本预算；
- 模型具体版本；
- 评分规则；
- 留出测试任务。

任何已经用于修改 Skill 的案例都属于开发数据。使用 [`score_run.py`](../.agents/skills/decision-research/scripts/score_run.py) 汇总不同实验条件的评分。

只有在修改工作流或与其他研究系统比较时，才按需读取 [前人方案索引](../.agents/skills/decision-research/references/prior-art.md)。
