### 2. 推理 — 交互失败在哪层？

**核心交叉发现：NOMI vs 小鹏 = 两条产品路线，同一个技术底座**

```
小鹏路线：全时对话默认持续收音 → 漏拒识率目标"万分之几" → 敢给
NOMI路线：全舱免唤醒手动开启   → 缩窄为仅主驾         → 不敢给
```

**这不是技术差距，是产品信心差距：**

| 对比维度 | 小鹏 | NOMI |
|---------|------|------|
| 拒识/漏拒识目标 | 万分之几（≈0.01%） | 96.8%准确率（≈3.2%错误率） |
| 免唤醒默认状态 | 开启（全时对话持续收音） | 关闭（需手动去设置翻开关） |
| 多音区策略 | 多人对话+全时对话可同时开启 | 全舱免唤醒→缩窄为仅主驾 |
| 产品哲学 | "把方便带给用户，把难题留给研发" | "如果连我自己这一关都过不去，我会让它多停留一段时间" |

两条路都"负责"，但在行业免唤醒渗透率>60%的2025年，NOMI的保守已经从"谨慎"滑向"落后"。

**技术定位：问题不在ASR/NLU层，在产品策略层。** NOMI有行业领先的多模拒识（OMS视觉+座椅传感器+Wav2Vec+TinyBert），但96.8%的准确率被用来论证"还不够好"，而小鹏用万分之几的目标倒逼技术——差了约300倍的目标设定，导致完全不同的默认体验。
