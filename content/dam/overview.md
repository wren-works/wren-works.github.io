# Design Memory

> 语义检索 × 设计资产管理 · 功能原型

---

## 一句话叙事

用自然语言找回“记得画面，却忘了文件名”的设计参考。验证语义检索是否比文件夹和关键词更符合设计师的记忆方式。

---

## 问题

设计师的日常：在 Figma 中做过一个按钮样式、在 Dribbble 存过一张参考图、在本地有一个图标库。但当需要复用某个资产时，只能靠记忆翻文件夹——"那个蓝色的按钮是放在哪个项目里的来着？"

现有工具的检索方式仍然是关键词匹配 + 文件夹层级，而设计师的思维方式是**视觉特征 + 使用场景**——"那个半透明磨砂效果的卡片"比"glass-card.png"更接近大脑的检索方式。

---

## 核心假设

1. **语义检索 vs 关键词检索**：自然语言描述（"暖色调的登录页"）能否比文件名搜索更快找到目标资产？
2. **多模态索引**：同时用文本、颜色、视觉特征索引同一个资产，检索体验是否更自然？
3. **浏览式发现**：不搜索，而是通过相似内容和颜色浏览资产库，是否符合设计师“逛灵感”的行为模式？

---

## 已实现的功能原型

- 上传与导入设计素材
- 使用向量索引进行自然语言语义检索
- 按图片、组件与颜色浏览素材库
- 查看图片详情、主色与基础视觉信息
- 使用 SQLite 保存元数据、Chroma 保存向量索引

完整原型当前依赖本地 Python API，暂未部署为公开在线服务。

---

## 已知约束

- 个人项目，无团队协作场景
- 资产规模受限（取决于个人设计文件积累）
- 语义模型选型需在本地推理和云端 API 之间权衡
- 目前完成的是个人场景功能原型，尚未进行设计师用户测试
- 当前素材以 Material Design 组件与测试图片为主，对真实个人设计资产的覆盖仍有限
