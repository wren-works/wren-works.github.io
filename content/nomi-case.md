# NOMI 车机AI评估 Loop

## Hero
用 AI 子代理接力跑了六轮 NOMI 车机分析。本想验证一个「设定好就能自己跑到成功」的 Loop 工作流——结果发现不是魔法。

## Context

打算写一份车机AI的产品分析当作品。

然后，摆烂了。

过了一阵在微信公众号看到 Loop 的概念，说得挺神的——写好限制和目标，AI 就能自己一直跑到成功。

说实话，心动了。

听起来像一个永远不会累的研究员，你只需要告诉它第一轮搜什么，它就能自己一圈一圈转下去，每轮都比上一轮挖得更深。

我决定拿 NOMI 试试。反正分析内容本身也够得上蔚来的 PM JD，跑歪了也不算全亏。

跑完六轮。发现这不是魔法。Loop 确实能接力——每一轮都在修正或叠加前一轮的发现，不是原地打转。但它需要人在岔路口把方向盘。第一轮差点就跑歪了。

## Approach

当时脑子里有两个东西想验证。

一是 Loop 本身——子代理到底能不能「读上次日志→自己搜→自己分析→自己定下一轮方向」。二是 NOMI 的产品策略到底是什么。后一个问题刚好适合迭代：表面的答案很容易（「它没免唤醒，落后」），但多轮挖下去能碰到根因。

设计上不复杂。每轮子代理做五件事：看上一轮的结论→搜新资料→提取案例→交叉分析→写总结并定下一轮搜索方向。关键在那个「定方向」——不是人类预设好所有路径，是子代理自己判断：看了这轮发现之后，接下来该搜什么。

然后问题就来了。

子代理有个毛病：一旦在早期轮次形成了猜测，后面就倾向于验证而非质疑。Loop #1 判断「NOMI 不支持免唤醒」，Loop #2 靠一条 NIO 官网版本说明就推翻了——但这个推翻不是因为子代理足够聪明去质疑自己，是碰巧搜到的。如果搜索方向偏了呢？后面四轮可能就沿着一条错的假设跑到底了。

这就是后来搁置的原因。它需要一个能在关键节点说「等等，这方向不对」的机制，而目前这个机制主要靠我。

## Solution

六轮，核心工作在一天跑完。

Loop #1，扔了五条已知链接进去。结论：NOMI 免唤醒「缺失」。看起来挺合理的，毕竟凤凰网四车横评里它确实是唯一不支持的。

Loop #2，翻车了。搜到 NIO 官网 Banyan 3.0.0 版本说明——功能有，但需要手动去设置里开，而且只给主驾用。

问题变了。不是「做没做到」。是「做到了为什么不给」。

后面开始补竞品对标。Loop #3 看小鹏——激进，免唤醒默认全开，漏拒识率「万分之几」。Loop #4 看理想——走中间，预定义高频指令默认响应。NOMI 呢？藏在设置里。

拼图到第五轮开始对上了。

NOMI 不是技术不行。96.8% 的多模拒识准确率不输竞品。问题是它不能像竞品那样「万一误响就算了」。用户给 NOMI 织帽子、起名字，把它当伙伴。同样的误响应，在小鹏上是功能瑕疵，在 NOMI 上是「它不懂我」——信任成本被情感定位放大了不知道多少倍。

第六轮收尾。产出了四份文档：评估框架（客观指标+信任度主观评估+场景矩阵）、体验方案（分层默认策略，方向盘按键做信任退路）、大模型分析和行业趋势。

## Reflection

NOMI 这边的材料是能用的。四份文档覆盖了 PM JD 的四个能力维度，核心叙事也完整：「保守不是技术落后，是品牌策略的代价。」面试官追问的话也有应对方案。

Loop 这边的结论更诚实一些。

它有三个硬伤。第一，没有信源置信度分析——抓到什么搜什么。第二，消息没有时间权重——2023 年的吐槽和 2025 年的技术更新，它一视同仁。第三，也是最大的问题：确认偏误。子代理容易沿着早期猜测一路跑，后面的搜索变成给自己找证据，而不是找真相。

但它也真的证明了子代理接力这条路走得通。六轮，没有一轮是废的，每一轮都在前一轮基础上推进了。

不是魔法。是可用的工具，需要配上会开的人。

最近打算捡起来改。还有别的东西想用 Loop 调研——这次加纠偏机制进去。
