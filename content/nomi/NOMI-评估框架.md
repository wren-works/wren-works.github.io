## 二、NOMI独有的评估维度

NOMI的核心差异化是「情感伙伴」而非「功能助手」。同一指标，含义不同：

| 指标 | 功能助手（理想） | 情感伙伴（NOMI） |
|------|---------------|----------------|
| 误唤醒 | "功能不稳定" | 「它不懂我」——人格背叛 |
| 漏拒识 | "偶尔犯傻" | 「它在不该说话时插嘴」 |
| 叫不醒 | "坏了" | 「它不理我了」 |

**框架原则**：不因为情感定位降低功能标准，但也不因为跑分忽视信任维度。

## 三、三个关键指标

### 3.1 拒识准确率——拆解才有判断力

96.8%（NOMI多模拒识）不能只看表面。必须拆成：

```
拒识准确率 96.8%
   ├── 漏拒识率（该拒却响应）→ NOMI用户感知："它插嘴/乱说"
   └── 误拒识率（该响却拒）  → NOMI用户感知："叫不醒"
```

| | NOMI | 理想 | 小鹏 |
|---|------|------|------|
| 拒识总准确率 | 96.8% | 95% Backchannel拒识 | — |
| 漏拒识率目标 | **需实测** | — | **万分之几 ≈ 0.01%** |

假设每天100次语音输入（含噪声/闲聊），3.2%错误=每天3.2次。小鹏"万分之几"=每100天1次。**不是技术差距，是指标设定的产品信心差距。**

### 3.2 用户信任度——跑分表量不到的维度

竞品测"做对了没"，NOMI还需要测"有没有让人感到被懂"：

| 信任维度 | 定义 | 可测量信号 |
|---------|------|-----------|
| **理解信任** | 它听懂我了吗？ | 首次响应成功率 |
| **人格信任** | 它不会在不该说话时插嘴？ | 漏拒识率 + "乱插嘴"投诉率 |
| **可靠信任** | 关键指令一次做对？ | 车控/导航类FCR |
| **边界信任** | 不确定时主动问我？ | 拒识后反问澄清比例 |

### 3.3 情感信任侵蚀——NOMI独有的预警机制

NOMI的信任一旦受损，修复成本高于功能型助手：

| 预警级别 | 触发条件 | 动作 |
|---------|---------|------|
| 🟢 正常 | 信任度 ≥ 4.0 | 常规追踪 |
| 🟡 关注 | 信任度 3.5-3.9 或 "不懂我"类吐槽 ≥ 3条/周 | 定位场景，评估修复 |
| 🔴 危险 | 信任度 < 3.5 或 大规模吐槽 | 启动信任修复专项 |
| ⚫ 危机 | 安全事件 | 策略性回撤（参照3.0→3.1缩窄免唤醒） |

## 四、场景分层测试矩阵

不测全场景的跑分是废纸。六维正交：

| 维度 | 分层 |
|------|------|
| 驾驶状态 | 静止 / 低速 / 高速 |
| 车内人数 | 单人 / 双人 / 全家 |
| 音区 | 主驾 / 副驾 / 后排 |
| 指令类型 | 车控 / 导航 / 媒体 / 闲聊 / 复合 |
| 交互模式 | 唤醒后 / 免唤醒 / 连续对话 |
| 噪声条件 | 静音 / 音乐 / 开窗 / 儿童 |

**三个最暴露问题的场景**（直接面试能讲）：

- **全家出行 + 免唤醒 + 后排闲聊**：验证全舱免唤醒缩窄为仅主驾的根因
- **高速 + 复合指令**：验证ASR+NLP在噪声下的全链路表现
- **副驾聊天 + 主驾指令**：验证多音区隔离+拒识的耦合

## 六、如何在面试中讲

**面试官问"你怎么评测一个AI产品"时的四段式回答：**

> **第一段（30秒）——框架**  
> "我分五层：ASR→NLU→对话管理→TTS→端到端。客观指标告诉你做对了没，主观指标告诉你用户觉得好不好。但更关键的是场景分层——高速上的WER和静音车库里完全不一样。"
>
> **第二段（30秒）——差异化**  
> "但对NOMI，通用框架不够。因为NOMI是情感型AI——同一个误响应，在理想同学上是功能不稳定，在NOMI上是'它不懂我'。所以我额外加了用户信任度评估和情感信任侵蚀预警。"
>
> **第三段（60秒）——具体案例**  
> "比如拒识准确率96.8%。表面看比理想95%还略高。但小鹏的漏拒识率目标是万分之几，差约300倍。不是技术差距，是产品信心差距。所以我不会只看96.8%就下结论——拆成漏拒识率和误拒识率，在多人+免唤醒的场景下实测，才能回答'NOMI能不能默认开'。"
>
> **第四段（30秒）——方法论收束**  
> "好的评估框架不是指标越多越好——是每个指标都能回答一个产品决策问题。'漏拒识率'回答能不能开免唤醒，'用户信任度'回答情感定位成不成立。指标不驱动决策，就只是数字。"

**常见追问的底牌：**

- "你怎么知道阈值定得合理？" → "三层参照：竞品数据、行业基准、用户容忍度。阈值每次OTA校准。"
- "数据从哪来？你没车怎么测？" → "四个来源按优先级：云端日志（大规模）→实车路测（因果）→实验室录音回放（AB对比）→用户反馈（主观校准）。诚实说我现在没实车数据——但这个框架的价值不在数据，在'数据该怎么收集和解读'。"
- "你觉得自己哪里没考虑到？" → "三个盲区：①多语言/方言我没设计case；②极端天气下车内声学我没有；③用户学会'怎么和NOMI说话更容易'之后，FCR会虚高。"
