# NOMI 体验方案：全舱免唤醒渐进式默认开启

> 场景A · 面试展示版 · 基于五轮Loop积累

---

## 一、场景

一个 NOMI 车主每天通勤。上车后想开空调、设导航、放音乐——三件每次都要先说"Hi NOMI"。他知道有个"全舱免唤醒"功能，但从没去设置里找过；朋友开了的说"不太灵光，副驾聊天老被误触发"。同一时间，隔壁理想车主坐进车里直接说"导航回家"。

**这不是功能缺失。NOMI 有免唤醒，但藏在手动设置里。** 三强对标：小鹏全时对话默认持续收音，理想预定义指令默认响应，NOMI 手动开启→缩窄为仅主驾。NOMI 连行业共识的"预定义指令默认开"都没做到。

**为什么没做？** 不是因为技术做不到——Wav2Vec+TinyBert 小字典天然支持预定义指令免唤醒。是因为 NOMI 的情感伙伴定位：同一个误响应，在理想同学上是"功能不稳定"，在 NOMI 上是"它不懂我"。产品团队用一刀切保守保护信任基础。

**但一刀切不是唯一选项。** 分层策略可以利用 NOMI 已有的技术分层（预定义指令层 + 开放域层），在确定性高处默认开、不确定性高处留给用户选。

---

## 二、体验目标

> 让用户坐上车就能自然说出高频指令，不用每次都先叫 NOMI 的名字——同时不让用户担心"它会不会在我聊天时乱插嘴"。

不是"把免唤醒全打开"，是**在信任保护下渐进式默认**。用户感知到的变化：不需要去设置找开关，也不需要担心误响应——因为只有确定性高的指令才默认免唤醒。

---

## 三、方案设计

### 3.1 分层默认策略

| 层级 | 指令范围 | 默认状态 | 为什么 |
|------|---------|---------|--------|
| **L1 高频车控** | 空调/车窗/座椅/方向盘加热等 30-50 条 | **默认开启**（全部音区） | 指令短、格式固定、NLU 置信度高，误响应风险极低 |
| **L2 导航+媒体** | 导航目的地/路线偏好/音乐控制/音量等 50-100 条 | **默认开启**（仅主驾音区） | 高频但歧义比 L1 多，先用主驾限定降低风险 |
| **L3 开放域对话** | 闲聊/问答/百科/情感对话 | **默认关闭，用户自选** | 开放域拒识难度高，误响应信任成本大——保留用户选择权 |
| **🆕 物理退路** | 任何误响应发生时 | 方向盘 NOMI 键 = 静默撤销 + 上报 | 不需要语音纠正，按一下就行，消除"乱插嘴"的无力感 |

### 3.2 为什么分三层而不是一刀切

- **L1 几乎零风险**：指令格式固定（"打开空调""温度调高"），Wav2Vec+TinyBert 小字典的置信度在 99% 以上。不默认开 L1 是产品信心的浪费
- **L2 副驾限定**：导航/媒体指令歧义略高（"回家"是哪个家、"放点音乐"放什么），用主驾音区限定 → 副驾聊天不会被误触发，降低 80% 的误响应场景
- **L3 用户自选**：开放域对话是理想都不敢默认开的领域。留给用户自选，尊重 NOMI 的信任定位

### 3.3 方向盘按键作为信任退路

这是 NOMI 独有的物理优势。理想/小鹏误响应后用户需要说"不是跟你说的"——语音纠错本身就是二次打扰。NOMI 的方向盘物理按键可以做：

> 用户按下 NOMI 键 → NOMI 静默撤销上一次免唤醒响应 + 上报为"误响应"日志 → 用户不需要说话纠正

按一下的成本远低于"翻设置开关"。这个退路降低了误响应后的用户无力感。

## 四、Success Metric

| 指标 | 目标值 | 为什么这个数 | 参照系 |
|------|-------|-------------|--------|
| **L1+L2 FCR（首次响应成功率）** | ≥ 95% | 比 96.8% 拒识准确率稍低，预留噪声余量 | 理想 96.5% 轮次准确率 |
| **漏拒识率（L1+L2）** | ≤ 0.1%（千分之一） | 小鹏万分之几是理想目标，千分之一是启动条件 | 小鹏 0.01% 目标 |
| **方向盘撤销率** | < 3/100 次 | 每百次免唤醒指令，用户按撤销 < 3 次 | 行业无参照，NOMI 首次定义 |
| **14 天留存率** | ≥ 80% 用户未主动关闭 L1/L2 | 默认开后两周内，不产生"被冒犯"到想关掉的程度 | 蔚来满意度体系 < 4.0 阈值 |

关键判断：**不追求 0 误响应。** — 追求的是误响应发生时用户不觉得被冒犯（有按键退路），且误响应频率低到不影响信任（千分之一级）。

## 五、验证方法

### 5.1 三阶段推进

```
内测（50人×2周）
  → 蔚来内部员工车主，收集 FCR + 漏拒识日志 + 满意度问卷
  → 硬门：L1+L2 FCR ≥ 92%，漏拒识率 < 0.3%，才能进公测

公测（500人×4周）
  → 蔚来App招募志愿者，NPS + 14天留存率 + 社区情感分析
  → 硬门：满意度 ≥ 4.0，留存率 ≥ 75%，无"乱插嘴"类吐槽爆发

全量（OTA推送）
  → 默认开启 L1 全部 + L2 主驾限定
  → 持续监控：云端日志自动追踪 FCR/漏拒识/撤销率
  → 触发回撤：满意度 < 3.5 或出现安全相关误触发 → 回退到内测版
```

### 5.2 特别针对 NOMI 的验证维度

- 不只测"做对了没"，还测**"用户有没有感到被冒犯"**：公测问卷包含"你觉得 NOMI 在你不跟它说话时插嘴了吗？"
- 方向盘撤销按键的使用日志：每个撤销事件都标记场景（车速/音区/车内人数/当前噪声来源），聚类分析哪些场景是漏拒识高发区
- 参照蔚来已有的满意度体系（4.4 优秀线）：L1+L2 默认开启后的满意度不得低于开启前

## 六、面试叙事

**如果面试官问"你怎么设计NOMI的一个功能体验"：**

> 「我选的是全舱免唤醒的渐进式默认开启。NOMI目前有免唤醒但需要手动去设置里翻开关，大部分用户根本不知道这个功能存在。但一刀切默认全开也不行——NOMI的情感伙伴定位让误响应的信任成本比竞品高很多。」
>
> 「所以方案是分三层：L1高频车控全部默认开（几乎零风险），L2导航+媒体仅主驾默认开，L3开放域留用户自选。同时利用NOMI独有的方向盘物理按键作为信任退路——误响应时按一下就撤销，不需要语音纠正。」
>
> 「验证分三阶段推进：内测→公测→全量，每个阶段有硬门指标。关键不追求0误响应，追求误响应发生时用户不觉得被冒犯，且频率低到不影响信任。」

## 已知局限

- 未验证 Wav2Vec+TinyBert 小字典的真实覆盖率：L1/L2 指令范围是基于行业经验的估计，NOMI 真实覆盖范围未知
- 方向盘撤销按键的交互需要和硬件团队对齐：是否复用现有 NOMI 唤醒键、长按/短按逻辑
- 蔚来满意度体系 < 4.0 的回撤触发线是我假设的，内部可能有不同标准
- 方案的基础假设（情感伙伴定位的双刃剑）来自公开信息推理，未与蔚来座舱 PM 验证
