## 一、技术架构

### 1.1 三层混合路由策略

NOMI GPT 采用"本地模型 → REJ Agent大模型 → 云端大模型"三层架构：

| 层级 | 部署位置 | 模型/技术 | 负责范围 | 延迟 |
|------|---------|----------|---------|------|
| **L1 本地** | 端侧8295芯片 | Wav2Vec + TinyBert | 预定义高频指令（车控/导航/媒体）、免唤醒指令 | <200ms |
| **L2 REJ Agent** | 端侧为主 | 认知中枢大模型 | 开放域上下文意图判断、对话指向、多模拒识 | 毫秒级 |
| **L3 云端** | 云端 | NOMI GPT大模型 | 百科问答、无限趣聊、情感对话、复杂推理 | 秒级 |

**核心路由逻辑：**
- **任务型指令**（"打开空调""导航到公司"）→ 本地NLU意图模型 + 技能模块直接执行，不走大模型推理
- **对话型问答**（"周杰伦有哪些歌""上海有哪些好玩的公园"）→ REJ Agent判断后，调用云端大模型
- **模糊/多意图指令**（"我有点冷也有点困"）→ 大模型语义拆分 → 调度分发 → 多技能并行执行

**端云协同的独特设计：**
- NOMI GPT 是**行业首家纯端侧部署的多模态感知大模型**，无需联网即可识别数千种物体（车内物品、车外楼宇/动植物）
- 端侧模型保证隐私数据不出车（账号隔离、端到端加密），云端模型提供无限知识
- 异构算力调度：骁龙8295 NPU（30TOPS）+ Orin-X跨域共享算力

**所以对NOMI意味着什么：** 这套架构让NOMI在"响应快"（车控毫秒级）和"懂得多"（云端无限知识）之间取得平衡。相比竞品全部依赖云端，NOMI的端侧能力在隧道/地库等无网环境仍然可用，这是体验上的护城河。

### 1.2 2000+技能的调度分发体系

认知中枢是NOMI GPT的"大脑皮层"，包含五大模块：

```
认知中枢
├── 调度分发     ← 核心：智能识别用户需求，调用对应服务
├── 多模拒识     ← 区分车内交谈 vs 对NOMI说话
├── 跨域继承/跳转 ← 多轮对话中保持上下文，跨功能域切换
├── 多Agent仲裁  ← 意图模糊时由Agent裁决真实需求
└── 知识库检索   ← 品牌知识/用车知识/生活百科/时效性内容
```

**技能调度流程（以"工作日早上导航到公司+开空调+放音乐"为例）：**
1. **NLU意图模型**→ 判断为"生成快捷场景"请求
2. **NOMI GPT大模型**→ 语义拆分：触发条件（工作日/上车后）+ 执行动作（导航/空调/音乐）
3. **调度分发**→ 整合为系统可执行指令，用户确认后持久保存
4. **条件触发执行**→ 座椅判断上车 + 时间判断工作日 → 并行分发给导航、空调、媒体服务

**两大技术创新：**
- **模态内外多维度对比学习**：在文本分类中引入对比学习，构建自动训练pipeline，语义区分能力大幅提升
- **大模型感知与推理融合**：单轮/多轮对话均保持高度准确性，语义理解准确率提升12%+

**所以对NOMI意味着什么：** 2000+技能不是堆叠菜单，而是通过调度中枢实现了"一句话调度多个技能"的编排能力。这比传统"一个指令一个功能"的交互效率高一个量级。

## 二、NOMI Agents 多智能体

### 2.1 架构设计

NOMI Agents 是**端云融合的多智能体框架**，包含六大Agent + 底层协作机制：

```
                    ┌──────────────┐
                    │  认知中枢    │ ← 统一调度与仲裁
                    └──────┬───────┘
           ┌───────────────┼───────────────┐
           ▼               ▼               ▼
    ┌──────────┐    ┌──────────┐    ┌──────────┐
    │ 知识类   │    │ 生成类   │    │ 任务类   │
    ├──────────┤    ├──────────┤    ├──────────┤
    │·用车问答 │    │·表情生成 │    │·探索发现 │
    │·大模型百科│    │·专属歌单 │    │·停车助手 │
    └──────────┘    └──────────┘    │·守卫摘要 │
                                    │·服务管家 │
                                    │·旅行回忆 │
                                    │·NOMI DJ  │
                                    └──────────┘
         ▲                              │
         └──────── 协作机制 ─────────────┘
              · 短期记忆 + 长期记忆（记住偏好与家人）
              · 规划（COT思维链 + Reflection反思）
              · 行动（车控/导航/媒体/预约服务）
```

**端云部署分工：**
- **端侧Agent**（停车助手、守卫摘要、探索发现）：与SkyOS深度融合，直接调用原子能力（车控硬件/软件），响应快、隐私安全
- **云端Agent**（大模型百科、服务管家）：连接更多云端工具资源接口，处理长距离上下文对话和复杂推理

### 2.2 体验影响

多智能体给NOMI带来的三个质变：

1. **从"单点功能"到"主动智能"**
   - 守卫摘要Agent：车辆被碰触/靠近 → 自动总结守护信息并主动告知用户
   - 服务管家Agent：根据事件（如保养到期）→ 自主预约服务，支持一语下单
   - 探索发现Agent：路过地标 → 主动介绍、回答车外事物疑问

2. **多任务并行与上下文保持**
   - 不同Agent负责不同域，不会出现"聊着天就忘了刚才的车控操作"
   - 跨域继承/跳转：从"上海公园推荐"聊到"播放周杰伦"，无缝切换
   - 多Agent仲裁：意图模糊时（"我有点冷"可能是调空调也可能是问天气），Agent协作裁决

3. **个性化体验的深化**
   - 记忆模块（短期+长期）记住用户偏好、家人喜好
   - 情感引擎驱动下，每个NOMI发展出与用户性格契合的"人设"
   - 六大Agent被定义为Banyan 3发布（2024.07），后续版本不断扩展

**所以对NOMI意味着什么：** NOMI Agents让座舱体验从"用户主动说"变为"NOMI主动做"。这与竞品的"更好的ChatGPT"路线形成差异——NOMI的目标不是更好的聊天机器人，而是"会帮你办事的Agent"。

## 三、大模型带来的体验质变

### 3.1 传统NLU vs 大模型

| 维度 | 传统NLU（2018-2023 NOMI） | 大模型NOMI（2024+） |
|------|--------------------------|---------------------|
| **交互模式** | 命令式："打开空调25度" | 对话式："今天好热啊" → 主动建议开空调 |
| **意图理解** | 规则匹配，固定句式 | 语义理解，同义表达自由 |
| **多轮对话** | 2-3轮，需保持同一域 | 跨域多轮，上下文持续保持 |
| **上下文记忆** | 单次会话，重新开始 | 短期+长期记忆，跨驾驶行程 |
| **知识范围** | 预设FAQs | 大模型百科，任意主题深入对话 |
| **任务编排** | 单一指令执行 | AI场景生成，一句话编排多步任务 |
| **情感表达** | 固定话术 | 生成式语音合成，情感音色适配 |

### 3.2 只有大模型能做的新能力

1. **无限趣聊**：不限定主题的开放式对话，做"倾诉树洞"
2. **角色扮演**：根据语境自动切换角色，NOMI可以是旅行向导/音乐DJ/用车专家
3. **AI场景生成**：自然语言描述场景，"每天下班上车自动导航回家+播放播客+空调23度" → 自动生成快捷场景
4. **情感引擎**：情境智能 + 专属智能 + 情感表达，根据上下文判断用户情绪并主动关怀
5. **超拟人情感音色**（Banyan 3.1.0，行业首个生成式语音合成框架）：根据文本内容自动适配喜怒哀乐情感变化，英语发音地道
6. **端侧多模态感知**：离线识别数千种物体、根据非颜色磁珠生成氛围灯颜色组合
7. **旅行回忆**：自动记录行程照片和路线，生成旅行回忆
8. **NOMI空间探索**：实时与窗外POI产生问答互动
9. **用车问答**：自然语言咨询车辆操作/故障，大模型理解后提供专业解答
10. **跨端指令**（Banyan 3.1.0）：通过NOMI语音控制手机微信/支付宝功能

### 3.3 新问题与NOMI的应对

| 问题 | NOMI的应对策略 |
|------|---------------|
| **幻觉** | ①多模拒识模块过滤不可靠输出；②认知中枢限定在"品牌知识/用车知识/生活百科/时效性"四大可信知识库内检索；③李天舒明确表态"幻觉和价值观对齐是最大的问题，在车里AI犯错的代价远比ChatGPT高"→ 不做不可靠的生成式能力（如AI变声、AI生图不第一时间做）|
| **延迟** | ①三层架构分级响应：高频车控走本地毫秒级；②端侧大模型无需联网即可完成多模态感知；③8295芯片30TOPS NPU + 舱驾融合跨域算力共享 |
| **成本** | ①端侧处理减少云端调用次数；②群体智能架构分摊训练成本；③SkyOS统一底层降低多车型适配成本 |
| **隐私** | ①端侧多模态感知，敏感信息不出车；②账号隔离、端到端加密；③李天舒："信任彻底崩塌是不可逆的" |
| **体验一致性** | ①小数点级五分制满意度（≥4.4优秀，<4.0严重不满）→ 精准定位问题功能；②Debug反馈系统（2018年建立）：用户在车上直接喊"Hi NOMI反馈问题"→ 几个小时内自动分发到产品经理 |

**所以对NOMI意味着什么：** 大模型引入后，NOMI的核心难题不是"能做多少事"，而是"不能做错事"。李天舒的产品哲学——"如果连我自己这一关都过不去，我会让它多停留一段时间"——决定了NOMI的能力扩张是克制的、体验优先的。这也是为什么NOMI没有第一时间上AI生图/AI变声等功能。

## 四、竞品大模型对比

| 维度 | NOMI GPT（蔚来） | MindGPT/MindGPT-3o（理想） | XGPT/天玑（小鹏） |
|------|---------|----------|------|
| **模型定位** | 端云多模态大模型 + 情感引擎 | 多模态认知大模型 | 融合通义+智谱的多模型方案 |
| **算力策略** | **端侧为主**（8295 30TOPS + Orin-X跨域共享），云端补充 | **云端+端侧协同**（让所有L系列车主都能用） | 云端大模型为主，车端蒸馏小模型 |
| **端侧能力** | ⭐⭐⭐ 离线识别数千种物体、端侧多模态感知 | ⭐⭐ 端侧NPU承担部分推理 | ⭐ 主要依赖云端 |
| **多智能体** | ✅ **6大Agent**（停车/守卫/管家/探索/DJ/旅行回忆），2024.07发布 | ⚠️ MindGPT-3o调用300+工具但不强调Agent化 | ⚠️ AI小P有推理能力，未明确多Agent框架 |
| **情感能力** | ⭐⭐⭐ **情感引擎**（情境智能+专属智能+情感表达），超拟人情感音色 | ⭐⭐ MindGPT-3o支持数十种语气词，但未强调情感引擎 | ⭐ 基础情感理解 |
| **语音合成** | **行业首个生成式语音合成框架**（Banyan 3.1.0） | 支持唱歌/动物叫等多种表达 | 标准TTS |
| **知识覆盖** | 行业最全知识库（品牌/用车/生活/时效性） | 111个领域、1000+专属能力 | 百科问答+联网搜索 |
| **隐私保护** | 端侧处理，数据不出车 | 云端处理，需联网 | 云端处理为主 |
| **硬件依赖** | 8295芯片（2024全系） | 兼容8155老车型 | 需新平台支持 |
| **核心差异** | "情感伙伴"：陪伴+主动服务 | "贾维斯"：实用工具+家庭场景 | "技术控"：大规模参数+快速迭代 |
| **迭代速度** | 130+版本迭代，2000+功能，35个OTA版本 | 每周一次MindGPT版本迭代 | 2天一次智驾模型迭代 |
| **产品哲学** | "如果连我自己这一关都过不去，会多停留一段时间" | "以终为始，每周迭代" | "大力出奇迹" |

### 路线差异总结

- **蔚来**：情感+隐私+主动智能路线。核心差异在于**端侧大模型能力**和**情感引擎**，让NOMI成为"懂你的情感伙伴"而非"更聪明的工具"。优势是离线可用、隐私安全、拟人化体验；代价是硬件门槛高（仅8295新车型可用）。
- **理想**：实用主义路线。MindGPT让所有L系列车主都能享受大模型，强调知识准确性和工具调用能力（美团/小红书/300+工具），更像一个"万能的贾维斯"。MindGPT-3o已进化为多模态智能体。
- **小鹏**：技术追赶路线。融合多个第三方大模型（通义/智谱），强调推理能力提升和快速迭代。AI小P已实现免唤醒、深度思考。但座舱大模型目前更多是"更好的ChatGPT"，缺少类似NOMI的端侧独立能力和情感体系。

## 五、面试叙事

### 5.1 如果面试官问"大模型怎么改变了车载语音"

> "大模型把车载语音从'命令执行器'变成了'认知伙伴'。以前你只能说'打开空调25度'，现在你可以说'今天好热啊'，NOMI会理解你的感受并主动调空调。这个变化的本质是交互范式从'人适应机器'变成'机器理解人'。
>
> 具体来说三个层面的质变：
> 第一，**理解力**——大模型让语音助手具备了跨域语义理解和上下文推理能力，不再是关键词匹配，而是真正理解意图。
> 第二，**行动力**——多智能体框架让语音助手从'听指令做事'变成'主动帮你想办法'，比如守卫摘要Agent在车辆被碰时会主动总结信息告知你。
> 第三，**情感力**——生成式语音合成让AI有了情绪表达能力，根据不同语境自动适配喜怒哀乐，这是传统TTS做不到的。
>
> 但大模型也带来了新挑战：幻觉、延迟、隐私。NOMI的做法是'分层架构'——高频车控走本地毫秒级、复杂对话走云端、端侧多模态保证数据不出车。以及'克制的能力扩张'——不做不可靠的生成能力，优先保证信任不崩塌。"

### 5.2 如果面试官问"NOMI在大模型上有什么独特优势"

> "NOMI在大模型上有三个其他车载AI难以复制的优势：
>
> **第一个是端侧大模型能力**。NOMI是行业首家纯端侧部署多模态感知大模型的，能在无网环境下识别数千种物体、根据物品颜色生成氛围灯。这意味着NOMI在隧道、地库、高速等无网场景仍然可用——对于一辆车来说，这是刚需。
>
> **第二个是情感引擎**。大多数车载大模型的目标是'更聪明的工具'，NOMI的目标是'有温度的情感伙伴'。情感引擎包含情境智能、专属智能和情感表达三个基石能力，配合记忆系统（短期+长期），每个NOMI都会发展出与用户性格契合的'人设'——这在行业中是独一无二的。
>
> **第三个是产品哲学**。李天舒说过'如果连我自己这一关都过不去，我会让它多停留一段时间'。NOMI的满意度体系是小数点级五分制，≥4.4才叫优秀，<4.0就是严重不满。这种对体验的极致追求让NOMI的能力扩张是克制的——不做AI变声因为涉及人设一致性，不抢上AI生图因为车载场景价值感不高。这种克制在'大力出奇迹'的大模型时代反而是稀缺的。
>
> 总结一句话：其他车载AI在比拼'能做多少事'，NOMI在回答'怎样让你更信任它'。"

## 六、已知局限

1. **硬件门槛高**：端侧大模型能力依赖8295芯片，仅2024款及以后新车型可用，老车主无法享受核心体验升级
2. **迭代速度相对慢**：130+版本迭代听上去多，但理想每周迭代一次MindGPT、小鹏2天迭代一次智驾模型——NOMI的"克制"哲学可能导致功能上新速度落后于竞品
3. **商业化验证不足**：NOMI Agents多智能体框架（2024.07发布）已近两年，但公开可查的用户满意度数据和功能使用率数据仍有限
4. **生态开放度**：相比理想接入美团/小红书、华为打通人-车-家全场景，NOMI的第三方服务生态接入还不明确
5. **智驾与座舱AI的融合**：理想已推进MindVLA（视觉-语言-动作统一），小鹏也在推世界基座模型，NOMI的"端云多模态大模型"在座舱领域领先，但舱驾融合的VLA方向尚未见明确产品
6. **成本压力**：端侧强算力 = 高硬件成本，蔚来均价35万+的高端定位锁死了用户基数，大规模数据飞轮效应不及比亚迪/理想
7. **情感引擎的"不可量化"**："更懂你的情感伙伴"是感性的产品定位，但在面试或商业场景中，缺乏可量化的硬指标对比（相比"识别准确率94%""推理速度30 Tokens/s"这类数字），需要准备好如何用体验案例代替数字竞争
